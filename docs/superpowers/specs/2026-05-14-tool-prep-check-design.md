# 사용 공기구와 준비물 점검 흐름 설계

## 목표

점검 메뉴에서 작업 유형을 선택한 뒤, 해당 작업에 필요한 사용 공기구와 준비물을 먼저 확인하고 체크한 항목에 맞춰 실제 점검 항목을 필터링한다. 공기구가 필요 없는 작업 유형은 준비물 확인 단계를 건너뛰고 기존처럼 바로 체크리스트로 이동한다.

## 범위

- 점검 메뉴에 `사용 공기구와 준비물` 중간 페이지를 추가한다.
- 더보기의 `섹션/항목 관리` 화면에서 작업 유형별 공기구/준비물 목록을 관리한다.
- 더보기의 기존 점검 항목 관리에서 이미 추가된 점검 항목 이름을 수정할 수 있게 한다.
- 점검 항목마다 관련 공기구/준비물을 연결하거나 연결하지 않을 수 있게 한다.
- 관리자 설정으로 공기구 체크 필수 여부를 ON/OFF 할 수 있게 한다.
- 더보기 화면에서 관리자가 픽토그램 라이브러리에 새 픽토그램을 추가하거나 기존 픽토그램을 삭제할 수 있게 한다.

## 사용자 점검 흐름

1. 사용자는 `점검` 메뉴에 진입한다.
2. 작업 유형 카드 중 하나를 선택한다.
3. 선택한 작업 유형에 등록된 공기구/준비물이 있으면 `사용 공기구와 준비물` 페이지로 이동한다.
4. 선택한 작업 유형에 등록된 공기구/준비물이 없으면 바로 체크리스트 페이지로 이동한다.
5. `사용 공기구와 준비물` 페이지에서 사용자는 실제 사용할 공기구/준비물을 체크한다.
6. 관리자 설정이 `체크 필수 ON`이면 1개 이상 체크해야 `다음 점검으로` 이동할 수 있다.
7. 관리자 설정이 `체크 필수 OFF`이면 체크 없이도 `다음 점검으로` 이동할 수 있다.
8. 체크리스트 페이지에는 다음 항목이 표시된다.
   - 관련 공기구가 없는 공통 점검 항목
   - 사용자가 체크한 공기구와 연결된 점검 항목
9. 체크하지 않은 공기구에만 연결된 점검 항목은 표시하지 않는다.

## 관리자 관리 흐름

`더보기 > 섹션/항목 관리`에서 작업 유형 하나를 선택하면 기존 관리 화면에 다음 관리 영역을 둔다.

### 아이콘/픽토그램 수정

기존 기능을 유지한다. 작업 유형 카드와 점검 화면에서 쓰는 픽토그램을 변경한다.

### 픽토그램 라이브러리 관리

아이콘/픽토그램 수정 영역 안에 관리자 전용 라이브러리 관리 기능을 추가한다.

- 픽토그램 추가
- 픽토그램 이름 수정
- 픽토그램 삭제
- 픽토그램 미리보기

픽토그램 추가는 이미지 파일을 선택해 등록하는 방식으로 설계한다. 등록된 이미지는 브라우저 저장소에 데이터 URL로 보관하고, 작업 유형의 `icon` 값은 해당 픽토그램 ID를 참조한다.

기본 제공 픽토그램은 삭제하지 못하게 한다. 관리자가 직접 추가한 사용자 픽토그램만 삭제할 수 있다. 삭제하려는 픽토그램이 현재 작업 유형에서 사용 중이면 삭제 전에 안내하고, 삭제 시 해당 작업 유형은 기본 픽토그램으로 되돌린다.

### 공기구/준비물 관리

아이콘/픽토그램 수정 영역 아래에 새 영역을 추가한다.

- 공기구/준비물 추가
- 공기구/준비물 이름 수정
- 공기구/준비물 삭제
- 공기구 체크 필수 ON/OFF

공기구/준비물은 작업 유형별 데이터로 관리한다. 예를 들어 `탑재 점검`의 공기구 목록과 `압력테스트`의 공기구 목록은 독립적이다.

### 섹션 관리

기존 섹션 추가/수정/삭제 흐름을 유지한다.

### 점검 항목 관리

기존 항목 추가/삭제/위험도 관리를 유지하면서 다음을 추가한다.

- 이미 추가된 점검 항목 이름 수정
- 점검 항목 위험도 수정
- 관련 공기구/준비물 연결/해제

관련 공기구를 하나도 선택하지 않은 점검 항목은 공통 점검 항목으로 간주하고 항상 표시한다.

## 데이터 모델

현재 앱은 브라우저 상태와 저장소에 다음 성격의 데이터를 갖고 있다.

- `categories`: 작업 유형
- `sections`: 작업 유형별 섹션
- `items`: 점검 항목
- `inspections`: 점검 이력

새 기능을 위해 다음 필드를 추가한다.

### 픽토그램 라이브러리

새 컬렉션 `pictograms`를 추가한다.

필드:

- `id`: 고유 ID
- `label`: 픽토그램 이름
- `src`: 이미지 데이터 URL 또는 기본 제공 이미지 경로
- `source`: `builtIn` 또는 `custom`
- `order`: 표시 순서
- `deleted`: 삭제 여부

기본 제공 픽토그램은 현재 `PICTOGRAMS`와 `PICTOGRAM_ASSETS`를 기반으로 초기화한다. 사용자 추가 픽토그램은 `source: custom`으로 저장한다.

### 작업 유형별 공기구/준비물

새 컬렉션 `tools`를 추가한다.

필드:

- `id`: 고유 ID
- `categoryId`: 작업 유형 ID
- `name`: 공기구/준비물 이름
- `order`: 표시 순서
- `deleted`: 삭제 여부

### 작업 유형 설정

작업 유형 데이터에 설정 필드를 추가한다.

- `requireToolCheck`: 공기구/준비물 체크 필수 여부

기본값은 `true`로 둔다. 단, 공기구/준비물이 없는 작업 유형은 이 값과 관계없이 준비물 페이지를 건너뛴다.

### 점검 항목과 공기구 연결

점검 항목 데이터에 다음 필드를 추가한다.

- `toolIds`: 이 점검 항목과 연결된 공기구/준비물 ID 목록

기본값은 빈 배열이다. 빈 배열이면 공통 점검 항목으로 처리한다.

### 점검 작성 상태

작성 중 상태에 다음 값을 추가한다.

- `selectedToolIds`: 사용자가 이번 점검에서 체크한 공기구/준비물 ID 목록
- `toolPrepComplete`: 공기구/준비물 페이지를 통과했는지 여부

작업 유형을 바꾸거나 뒤로 돌아가면 이 값은 초기화한다.

## 필터링 규칙

체크리스트에 표시할 항목은 다음 규칙으로 계산한다.

- `item.toolIds`가 비어 있으면 항상 표시한다.
- `item.toolIds` 중 하나라도 `selectedToolIds`에 포함되어 있으면 표시한다.
- `item.toolIds`가 있지만 체크한 공기구와 겹치지 않으면 숨긴다.

이 규칙은 점검 화면의 진행률, 위험 항목 개수, 제출 가능 여부 계산에도 동일하게 적용한다.

## 점검 이력 저장

점검 제출 시 다음 정보를 이력에 함께 저장한다.

- 선택한 공기구/준비물 이름 목록
- 실제 표시되어 점검한 항목 목록

이렇게 하면 나중에 공기구 선택 기준이 바뀌어도 과거 이력은 제출 당시 상태를 유지한다.

## 화면 설계

### 사용 공기구와 준비물 페이지

상단:

- 작업 유형명
- 설명 문구: `사용할 공기구와 준비물을 체크한 뒤 다음 점검으로 이동하세요.`
- 뒤로 버튼

본문:

- 공기구/준비물 체크 카드 목록
- 체크된 개수 표시
- 공기구 체크가 필수인데 0개 선택된 경우 안내 문구 표시

하단:

- `다음 점검으로` 버튼

### 더보기 관리 화면

작업 유형 상세 관리 화면의 오른쪽 패널에 다음 순서로 배치한다.

1. 아이콘/픽토그램 수정
2. 픽토그램 라이브러리 관리
3. 공기구/준비물 관리
4. 공기구 체크 필수 ON/OFF

기존 섹션/항목 목록에는 각 점검 항목 행 안에 항목명 수정과 관련 공기구 선택 UI를 추가한다.

## 예외 처리

- 공기구/준비물이 없는 작업 유형은 준비물 페이지를 표시하지 않는다.
- 공기구 목록은 있지만 연결된 점검 항목이 하나도 없어도 공통 점검 항목은 표시한다.
- 공통 점검 항목도 없고 표시할 항목이 없으면 `표시할 점검 항목이 없습니다. 더보기에서 항목을 연결하거나 공통 항목을 추가하세요.` 안내를 표시한다.
- 삭제된 공기구가 기존 점검 항목에 연결되어 있으면 화면 계산에서는 무시한다.
- 삭제된 공기구 이름은 과거 점검 이력에는 보존한다.

## 검증 기준

- 공기구가 없는 작업 유형 선택 시 바로 체크리스트가 열린다.
- 공기구가 있는 작업 유형 선택 시 준비물 페이지가 먼저 열린다.
- 체크 필수 ON 상태에서 아무것도 체크하지 않으면 다음 버튼이 비활성화된다.
- 체크 필수 OFF 상태에서는 아무것도 체크하지 않아도 다음으로 이동한다.
- 체크한 공기구와 연결된 항목만 표시되고, 공통 항목은 항상 표시된다.
- 더보기에서 공기구 추가/수정/삭제가 가능하다.
- 더보기에서 관리자가 사용자 픽토그램 추가/수정/삭제를 할 수 있다.
- 기본 제공 픽토그램은 삭제할 수 없다.
- 더보기에서 기존 점검 항목 이름 수정이 가능하다.
- 더보기에서 점검 항목별 관련 공기구 연결/해제가 가능하다.
- 점검 제출 이력에 선택한 공기구/준비물이 보존된다.
