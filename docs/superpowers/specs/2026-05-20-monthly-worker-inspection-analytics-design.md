# Monthly Worker Inspection Analytics Design

## Goal

통계 메뉴에 `월간 작업자 점검 현황` 섹션을 추가한다. 관리자가 한 달 기준으로 작업자들이 매일 작업 전 점검을 잘 수행했는지 빠르게 확인할 수 있어야 한다.

승인된 시각 기준은 `visual-verification/monthly-worker-analytics-5-options.svg`의 5번 혼합형이다.

## User Value

- 월간 점검률을 한눈에 확인한다.
- 오늘 미점검 작업자를 즉시 파악한다.
- 3일 이상 누락된 작업자를 우선 확인한다.
- 작업자별로 어느 날짜에 점검이 빠졌는지 패턴을 본다.

## Screen Placement

대상 화면은 `통계` 메뉴의 `renderAnalyticsDashboard()` 결과 화면이다.

새 섹션은 기존 KPI 카드 아래, `호선 공정 현황 / 위험도 분포` 섹션 위에 배치한다. 이유는 월간 작업자 점검 현황이 통계 화면의 주요 운영 지표이므로 최근 활동보다 먼저 보여야 하기 때문이다.

섹션 제목:

`월간 작업자 점검 현황`

보조 문구:

`2026년 5월 · 작업자별 일일 점검 이행 현황`

## Approved Layout

섹션은 세 부분으로 구성한다.

1. 월간 요약 KPI
   - 월간 점검률
   - 오늘 미점검
   - 3일 이상 누락
   - 대상 작업자

2. 작업자 x 날짜 히트맵
   - 행: 작업자
   - 열: 월의 날짜
   - 색상:
     - 녹색: 해당 날짜 점검 완료
     - 주황: 일부 완료 또는 미완료 기록 있음
     - 빨강: 대상일인데 점검 기록 없음
     - 회색: 미래일 또는 대상 제외일

3. 주의 필요 작업자
   - 월간 누락일이 많은 작업자를 우선 표시
   - 기본 기준은 누락 3일 이상
   - 각 카드에는 작업자명, 소속/팀, 누락일 수, 최근 누락일을 표시

## Data Source

기존 로컬 상태를 사용한다.

- `state.workers`
- `state.inspections`
- 신규 로컬 상태 `monthlyWorkerRestDays`

Supabase 테이블은 현재 동기화된 `safety_workers`, `safety_inspections`의 로컬 반영값을 그대로 사용한다. 휴무일 설정은 1차 구현에서 로컬 저장소 기준으로 관리한다. 원격 동기화가 필요해지면 별도 Supabase 테이블을 추가 설계한다.

## Month Scope

초기 구현은 현재 KST 기준 월을 기본으로 열고, 월 이동 버튼으로 이전 달과 다음 달을 탐색할 수 있게 한다.

- 기준 날짜는 기존 `serverNow()`, `today()`, `localDate()` 흐름을 따른다.
- 선택 월 시작일은 해당 월 1일이다.
- 선택 월 종료일은 해당 월 마지막 날이다.
- 미래 날짜는 회색 대상 제외로 표시한다.

월 이동 컨트롤:

- `이전 달`
- `이번 달`
- `다음 달`

`다음 달`은 현재 월보다 이후로 이동하지 못하게 비활성화한다. 과거 월은 점검 이력이 있는 범위까지 탐색 가능하게 한다. 별도 이력이 없어도 빈 상태를 표시한다.

## Worker Eligibility

대상 작업자는 `state.workers`에 등록된 작업자 전체다.

작업자가 등록되지 않았지만 점검 이력에는 이름이 있는 경우, 해당 이름도 보조 작업자로 포함한다. 이 경우 소속은 `기록 기반`으로 표시한다.

삭제된 작업자라도 월간 점검 이력에 이름이 있으면 히트맵에는 표시한다. 기존 기록 해석을 위해 필요하다.

## Daily Status Rules

작업자별 날짜 상태는 다음 규칙으로 계산한다.

- 완료: 해당 작업자의 해당 날짜 점검 기록 중 `status === "완료"`가 1건 이상 있고, 완료율이 100%인 기록이 있다.
- 일부: 해당 날짜 점검 기록은 있지만 완료가 아니거나 완료율이 100% 미만이다.
- 누락: 해당 날짜가 오늘 이전 또는 오늘이고, 해당 작업자의 점검 기록이 없다.
- 휴무: 해당 작업자가 해당 날짜에 휴무일로 설정되어 있다.
- 제외: 해당 날짜가 미래 날짜다.

하루에 여러 건 점검한 경우 가장 좋은 상태를 우선한다.

우선순위:

`휴무 > 완료 > 일부 > 누락 > 제외`

휴무일은 점검 대상일에서 제외한다. 휴무일에 점검 기록이 있더라도 KPI 계산에서는 휴무를 우선한다. 화면에는 휴무 색상으로 표시하고, 필요하면 이후 상세 기능에서 해당일 점검 기록을 따로 보여준다.

## KPI Rules

월간 점검률:

`완료일 수 / 대상일 수`

대상일 수는 `대상 작업자 수 x 선택 월에서 오늘 또는 월 마지막일까지의 날짜 수`에서 휴무일과 미래일을 제외한 값이다.

미점검 KPI:

선택 월이 현재 월이면 카드 라벨은 `오늘 미점검`이고, 오늘 날짜 기준 `누락` 상태인 작업자 수를 표시한다.

선택 월이 과거 월이면 카드 라벨은 `월말 미점검`이고, 선택 월의 마지막 대상일 기준 `누락` 상태인 작업자 수를 표시한다.

3일 이상 누락:

선택 월에서 누락일 수가 3일 이상인 작업자 수다. 휴무일은 누락일로 계산하지 않는다.

대상 작업자:

월간 히트맵에 포함된 작업자 수다.

## Rest Day Settings

통계 섹션 안에 `휴무 설정` 버튼을 추가한다.

휴무 설정은 작업자별 날짜 단위로 관리한다.

- 작업자 선택
- 날짜 선택
- 휴무 추가
- 등록된 휴무 삭제

저장 구조:

```js
{
  "김준혁": ["2026-05-03", "2026-05-10"],
  "박성호": ["2026-05-12"]
}
```

저장 키:

`monthlyWorkerRestDays`

휴무일은 색상으로 별도 표시한다. 기본 색상은 회색 계열 또는 파란 회색 계열로 한다. 빨강 누락과 혼동되지 않아야 한다.

## Excel Export

통계 섹션 안에 `월간 내보내기` 버튼을 추가한다.

내보내기 대상은 현재 선택 월의 월간 작업자 점검 현황이다.

파일명:

`monthly-worker-inspections-YYYY-MM.xlsx`

시트명:

`월간작업자점검`

컬럼:

- 작업자
- 소속/팀
- 월간 점검률
- 완료일 수
- 일부일 수
- 누락일 수
- 휴무일 수
- 대상일 수
- 1일
- 2일
- ...
- 31일

날짜 컬럼 값:

- `완료`
- `일부`
- `누락`
- `휴무`
- `제외`

기존 `exportRecords("analytics")`는 그대로 두고, 월간 전용 내보내기 분기를 추가한다.

## Interactions

초기 구현 상호작용은 다음으로 제한한다.

- 히트맵 셀은 클릭하지 않는다.
- 작업자 행 또는 주의 필요 작업자 카드를 누르면 해당 작업자의 월간 상세 요약을 펼치는 기능은 이번 범위에 넣지 않는다.
- `이전 달`, `이번 달`, `다음 달` 버튼으로 선택 월을 변경한다.
- `월간 내보내기` 버튼으로 현재 선택 월 데이터를 엑셀로 내려받는다.
- `휴무 설정` 버튼으로 휴무일 설정 패널을 열고 닫는다.

이 기능은 통계 화면을 읽기 쉽게 만드는 1차 구현이다.

## Mobile Layout

모바일이 주 사용 화면이다.

- KPI는 2열 또는 1열로 반응형 배치한다.
- 월 이동 버튼과 `휴무 설정`, `월간 내보내기` 버튼은 줄바꿈 가능한 툴바로 배치한다.
- 히트맵은 작업자명이 고정 느낌으로 왼쪽에 보이고, 날짜 셀은 가로 스크롤을 허용한다.
- 셀 크기는 터치 대상보다는 시각 스캔을 우선한다.
- 주의 필요 작업자 카드는 히트맵 아래에 세로 목록으로 표시한다.
- 휴무 설정 패널은 히트맵 아래에 세로 폼으로 표시한다.

모바일에서 표 전체가 화면을 깨지 않도록 히트맵 컨테이너에만 `overflow-x: auto`를 적용한다.

## Desktop Layout

데스크톱에서는 승인 시안처럼 한 섹션 안에 넓게 배치한다.

- KPI 4개는 한 줄
- 히트맵은 왼쪽 넓은 영역
- 주의 필요 작업자는 오른쪽 영역
- 휴무 설정 패널은 버튼을 누른 경우 섹션 하단에 전체 폭으로 표시

## Empty States

작업자가 없으면:

`등록된 작업자가 없습니다. 관리 메뉴에서 작업자를 먼저 추가하세요.`

점검 이력이 없으면:

히트맵은 작업자 행과 누락 상태를 보여준다. 월간 점검률은 0%가 된다.

작업자도 없고 이력도 없으면:

섹션 전체에 빈 상태 메시지만 표시한다.

## Testing

최소 검증:

- `npm.cmd run verify`
- `tests/static-recovery.test.js`에 새 섹션 제목과 핵심 함수 존재 검증 추가
- 모바일 alias 검증 시 통계 화면에서 섹션이 깨지지 않는지 확인

수동 확인:

- 작업자가 없을 때 빈 상태
- 작업자가 있고 점검 이력이 없을 때 0%와 누락 표시
- 오늘 완료/미완료/누락 상태가 색상으로 구분되는지
- 미래 날짜가 회색 제외로 표시되는지
- 이전 달 / 이번 달 / 다음 달 이동이 선택 월을 올바르게 바꾸는지
- 현재 월에서 다음 달 버튼이 비활성화되는지
- 휴무 설정 시 대상일과 누락일에서 제외되는지
- 월간 내보내기 파일에 선택 월과 날짜별 상태가 포함되는지

## Out of Scope

- 작업자 상세 드릴다운
- 관리자 알림 발송
- 작업 유형별 대상 작업자 필터
- 휴무일 원격 동기화

위 항목은 1차 구현 이후 필요하면 별도 기능으로 설계한다.
