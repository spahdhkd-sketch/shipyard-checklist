# 홈 불안전 요소 카운트 연동 설계

## 목표

홈 화면의 기존 `위험 미확인` 통계 카드를 `불안전 요소`로 변경하고, 새로 등록된 불안전요소 중 관리자가 아직 확인하지 않은 건수를 보여준다.

이 카드는 점검 기록의 위험 경고 수가 아니라, 불안전요소 등록 업무의 접수 상태를 나타내는 운영 지표로 사용한다.

## 결정 사항

- 카드 이름은 `불안전 요소`로 변경한다.
- 보조 문구는 `즉시 확인 필요`로 유지한다.
- 카운트 기준은 `unsafeIssues` 중 상태가 `접수`인 기록 수다.
- 작업자가 불안전요소를 등록하면 새 기록은 기본 상태 `접수`로 생성되고 카운트에 포함된다.
- 관리자가 상태를 `조치중` 또는 `완료`로 변경하면 해당 기록은 카운트에서 제외된다.
- 홈 통계 카드를 누르면 관리 화면의 불안전요소 목록으로 이동한다.
- 이동 시 불안전요소 상태 필터는 `접수`로 맞춰 즉시 확인이 필요한 건만 먼저 보여준다.
- 불안전요소 목록은 관리자 모드가 아니어도 볼 수 있다.
- 상태 변경, 조치 메모 저장, 삭제는 관리자 모드에서만 가능하다.
- 기록 화면의 기존 `위험 미확인` 탭은 제거한다.

## 배경

현재 홈의 `위험 미확인` 카운트는 점검 기록의 `warnings` 합계를 사용한다. 하지만 점검 작성 화면에서는 위험 상태 항목을 확인하지 않으면 제출이 되지 않으므로, 기록 화면에서 별도 `위험 미확인` 탭을 유지할 필요가 낮다.

새로운 불안전요소 등록 흐름에서는 작업자가 현장에서 빠르게 위험 요소를 등록하고, 관리자가 이를 확인해 `접수`에서 `조치중`으로 넘기는 흐름이 핵심이다. 따라서 홈 통계 카드는 “아직 관리자 확인 전인 불안전요소 접수 건”을 보여주는 것이 더 명확하다.

## 사용자 흐름

1. 작업자가 홈의 `불안전요소 등록` 메뉴로 이동한다.
2. 호선, 등록자, 내용, 사진을 입력하고 등록한다.
3. 새 기록은 상태 `접수`로 저장된다.
4. 홈의 `불안전 요소` 카운트가 1 증가한다.
5. 사용자가 홈의 `불안전 요소` 통계 카드를 누른다.
6. 관리 화면의 불안전요소 목록으로 이동하고, 상태 필터는 `접수`로 적용된다.
7. 작업자도 목록은 볼 수 있지만 상태 변경 컨트롤은 사용할 수 없다.
8. 관리자가 관리자 모드로 진입한 뒤 상태를 `조치중`으로 변경한다.
9. 해당 기록은 홈 `불안전 요소` 카운트에서 빠진다.

## 화면 변경

### 홈 화면

기존 통계 카드:

- 이름: `위험 미확인`
- 카운트: 점검 기록의 위험 경고 합계
- 보조 문구: `즉시 확인 필요`

변경 후 통계 카드:

- 이름: `불안전 요소`
- 카운트: 상태가 `접수`인 불안전요소 건수
- 보조 문구: `즉시 확인 필요`
- 클릭 동작: 관리 화면의 불안전요소 목록으로 이동

### 관리 화면

홈 통계 카드에서 진입할 때는 다음 상태가 적용된다.

- `manageTab`: `unsafe`
- `unsafeFilters.status`: `접수`
- `unsafeDetailId`: 비움

관리자 모드가 아니어도 불안전요소 목록은 표시한다. 단, 상태 선택, 조치 메모 저장, 삭제 버튼은 기존 관리자 권한 규칙을 따른다.

### 기록 화면

기록 화면의 범위 선택에서 `위험 미확인` 탭을 제거한다.

남는 탭:

- `전체 이력`
- `오늘 점검`
- `인도 예정`

기존 점검 기록의 위험 경고 수는 홈 통계에서 더 이상 사용하지 않는다.

## 데이터 규칙

불안전요소 카운트는 다음 규칙으로 계산한다.

```text
unsafeIssues.filter(row => row.status === "접수").length
```

상태값은 기존 불안전요소 상태 흐름을 그대로 사용한다.

```text
접수 -> 조치중 -> 완료
```

카운트는 `접수` 상태만 포함한다. `조치중`은 관리자가 이미 확인한 상태로 보고 카운트에서 제외한다.

## 테스트 방향

- 불안전요소 `접수` 기록이 3건이면 홈 `불안전 요소` 카운트가 `3건`으로 표시되는지 확인한다.
- 같은 기록 중 하나를 `조치중`으로 변경하면 홈 카운트가 줄어드는지 확인한다.
- 홈 통계 카드 클릭 시 관리 화면의 불안전요소 탭으로 이동하는지 확인한다.
- 진입 후 상태 필터가 `접수`로 적용되는지 확인한다.
- 기록 화면에 `위험 미확인` 탭이 더 이상 표시되지 않는지 확인한다.
- 관리자 모드가 아니어도 목록은 보이지만, 상태 변경과 삭제는 제한되는지 확인한다.

## 범위 제외

- 불안전요소 상태 변경 이력 타임라인은 이번 범위에 포함하지 않는다.
- 불안전요소 알림, 푸시, 배지 등 별도 알림 기능은 포함하지 않는다.
- 점검 기록의 위험 경고 저장 구조는 변경하지 않는다.
- Supabase RLS 정책은 이번 설계에서 변경하지 않는다.
