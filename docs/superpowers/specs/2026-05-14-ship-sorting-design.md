# 호선 메뉴 정렬 기능 설계

## 목표

호선 메뉴에서 등록된 호선 카드가 많아졌을 때 현장 사용자가 원하는 기준으로 빠르게 찾고 비교할 수 있도록 정렬 기능을 추가한다. 모바일 화면을 기준으로 조작이 간단해야 하며, 기존 호선 카드 레이아웃을 크게 흔들지 않는다. 관리자 모드에서는 현재 정렬 결과를 실제 저장 순서로 확정할 수 있게 한다.

## 현재 구조

- `renderShips()`가 호선 메뉴 전체를 렌더링한다.
- `renderProcessBoard()`는 공정 상태별 요약 보드를 표시한다.
- `renderShipRow()`는 개별 호선 카드를 표시한다.
- 현재 호선 정보 카드 목록은 `state.ships.map(renderShipRow)`로 표시되어 등록 순서에 가깝다.
- 작업자 점검 화면의 호선 목록은 `visibleWorkerShips()`에서 호선 번호순으로 정렬한다.

## 정렬 방식

호선 정보 카드 영역 상단에 모바일 친화적인 정렬 바를 추가한다.

정렬 옵션:

- `호선 번호순`: 호선 번호를 자연 정렬한다. 예: `H3449`, `H3512`
- `공정 상태순`: `탑재 > L/C > S/T > C/L > D/L` 순서로 묶고, 같은 상태 안에서는 호선 번호순으로 정렬한다.
- `L/C일 빠른순`: L/C일이 빠른 호선부터 표시하고, 미입력 호선은 뒤로 보낸다.
- `D/L일 빠른순`: D/L일 또는 인도일이 빠른 호선부터 표시하고, 미입력 호선은 뒤로 보낸다.
- `최근 추가순`: 새로 추가된 호선이 위로 오도록 표시한다.

기본 정렬은 `공정 상태순`으로 둔다. 현재 앱에서 호선 상태와 공정 색상을 강조하고 있으므로 현장 흐름과 가장 잘 맞는다.

## 화면 설계

`호선 정보 카드` 제목 줄 아래, 안내 문구 위 또는 안내 문구 바로 아래에 정렬 컨트롤을 둔다.

모바일 배치:

- 한 줄 상단: `정렬` 라벨 + 드롭다운
- 드롭다운은 전체 폭을 사용한다.
- 정렬 결과 개수는 기존 `호선 정보 카드 00척` 표시를 유지한다.

데스크탑 배치:

- 제목 줄 오른쪽 또는 안내 문구 아래 오른쪽 정렬로 배치한다.
- 별도 카드나 큰 영역을 만들지 않는다.

관리자 모드 추가 버튼:

- `현재 순서 저장` 버튼을 정렬 드롭다운 옆 또는 아래에 표시한다.
- 이 버튼은 수정 모드 ON 상태에서만 활성화한다.
- 버튼을 누르면 현재 화면에 보이는 호선 순서를 실제 `state.ships` 배열 순서로 반영하고 저장한다.

## 상태 저장

정렬 기준은 로컬 저장소에 저장한다.

- 키: `shipSortMode`
- 기본값: `stage`

사용자가 정렬 기준을 바꾸면 같은 기기에서 다시 접속해도 마지막 정렬 기준을 유지한다.

## 데이터 모델

호선 데이터에 새 필드를 강제 추가하지 않는다. `최근 추가순`은 기존 `order` 또는 배열 위치를 우선 사용한다. 기존 데이터에 `createdAt`이 있다면 `createdAt`을 우선 사용하고, 없으면 현재 배열 순서를 fallback으로 사용한다.

관리자가 `현재 순서 저장`을 누르면 `state.ships` 배열 자체를 현재 정렬 결과 순서로 재배열한다. 이 순서는 저장소에 반영되어 새로고침 후에도 유지된다.

필요한 앱 상태:

- `state.shipSortMode`

## 정렬 규칙

### 호선 번호순

- 문자열 자연 정렬을 사용한다.
- 숫자가 섞여도 `H9`보다 `H10`이 뒤에 오도록 처리한다.

### 공정 상태순

- `SHIP_WORKFLOW_STAGES` 순서를 기준으로 한다.
- 같은 공정 상태 안에서는 호선 번호순으로 정렬한다.

### L/C일 빠른순

- `lcDate`가 있는 호선이 먼저 온다.
- 날짜가 빠른 순으로 정렬한다.
- 날짜가 같으면 호선 번호순으로 정렬한다.
- 미입력 호선은 뒤에 표시한다.

### D/L일 빠른순

- `dlDate` 또는 `shipDeliveryDate(ship)` 기준으로 정렬한다.
- 날짜가 있는 호선이 먼저 온다.
- 날짜가 빠른 순으로 정렬한다.
- 날짜가 같으면 호선 번호순으로 정렬한다.
- 미입력 호선은 뒤에 표시한다.

### 최근 추가순

- `createdAt`이 있으면 최신순으로 정렬한다.
- `createdAt`이 없으면 현재 `state.ships` 배열에서 뒤에 있는 항목을 최신으로 간주한다.

## 이벤트 흐름

1. 사용자가 호선 메뉴에 진입한다.
2. 앱은 `state.shipSortMode` 또는 저장된 `shipSortMode`를 읽는다.
3. `sortedShips()` helper가 정렬된 호선 배열을 반환한다.
4. `renderShips()`는 `state.ships` 대신 `sortedShips()` 결과로 카드를 렌더링한다.
5. 사용자가 정렬 드롭다운을 변경하면 `state.shipSortMode`를 갱신하고 localStorage에 저장한 뒤 다시 렌더링한다.
6. 관리자 모드에서 `현재 순서 저장`을 누르면 `sortedShips()` 결과를 `state.ships`에 반영하고 `persist()`를 호출한다.
7. 저장 후 정렬 기준을 `저장된 순서` 또는 기본 표시 모드로 전환할 수 있다. 단순한 구현을 위해 첫 버전에서는 현재 선택된 정렬 기준을 유지해도 된다.

## 예외 처리

- 날짜가 잘못된 값이면 미입력과 동일하게 뒤로 보낸다.
- 호선 번호가 비어 있는 데이터는 기존 `dedupeShips()` 흐름에서 정리하므로 별도 표시하지 않는다.
- 정렬 드롭다운 변경만으로는 실제 `state.ships` 배열 순서를 변경하지 않는다.
- 관리자 모드의 `현재 순서 저장` 버튼을 누를 때만 실제 `state.ships` 배열 순서를 변경한다.
- 삭제, 날짜 수정, 공정 상태 수정은 기존 데이터에 적용되고, 렌더링 시 현재 정렬 기준으로 다시 표시된다.

## 검증 기준

- 호선 메뉴에 정렬 드롭다운이 표시된다.
- 기본값은 공정 상태순이다.
- 호선 번호순 선택 시 호선 번호 기준으로 자연 정렬된다.
- 공정 상태순 선택 시 `탑재 > L/C > S/T > C/L > D/L` 순서로 표시된다.
- L/C일 빠른순 선택 시 L/C일 미입력 호선은 뒤로 이동한다.
- D/L일 빠른순 선택 시 D/L/인도일 미입력 호선은 뒤로 이동한다.
- 최근 추가순 선택 시 새로 추가한 호선이 위에 표시된다.
- 정렬 기준은 새로고침 후에도 유지된다.
- 정렬 드롭다운 변경만으로는 데이터 저장 순서를 바꾸지 않는다.
- 관리자 모드에서 `현재 순서 저장`을 누르면 현재 화면 순서가 실제 저장 순서로 반영된다.
- 저장된 순서는 새로고침 후에도 유지된다.

## 설계 셀프리뷰

- 구현 범위는 호선 메뉴 표시 정렬, 로컬 설정 저장, 관리자 확정 저장에 한정했다.
- 새 데이터 필드를 강제하지 않아 기존 호선 데이터와 호환된다.
- 모바일 화면에서 드롭다운 하나로 조작하도록 단순화했다.
- 공정 상태 기반 앱 흐름에 맞춰 기본 정렬을 공정 상태순으로 정했다.
- 실수 방지를 위해 드롭다운 변경과 실제 데이터 순서 저장을 분리했다.
